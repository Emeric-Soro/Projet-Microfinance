4. SERVICE APICLIENT (Bridge avec l'API Java)
─────────────────────────────────────────────────

  src/Service/ApiClientService.php :
  - HttpClient vers http://localhost:8080/api
  - Gestion du JWT (login, refresh, blacklist)
  - Headers : Authorization, X-Correlation-Id, X-Idempotency-Key
  - Cache des réponses GET (cache Redis, TTL paramétrable)
  - Circuit breaker pour les appels longs
  - Mapping des erreurs API vers exceptions PHP

  Méthodes principales :
  - get(string $path, array $params = []) : ResponseInterface
  - post(string $path, array $data = []) : ResponseInterface
  - put(string $path, array $data = []) : ResponseInterface
  - delete(string $path) : ResponseInterface
  - login(string $user, string $pass) : JwtToken
  - loginOtp(string $user, string $challenge, string $otp) : JwtToken
  - refreshToken() : JwtToken

  ÉTAPE 2 — Coordonnées
    Champ                    Type      Validation                Requis
    ─────────────────────────────────────────────────────────────────
    Téléphone 1             tel       +225 format regex         OUI
    Téléphone 2             tel       format regex              NON
    Email                   email     format email              NON
    Adresse (région)        select    régions UEMOA             OUI
    Adresse (département)   select    filtré par région         OUI
    Adresse (commune)       text                                OUI
    Adresse (quartier)      text                                NON
    Boîte postale           text      5 chiffres                NON

  ÉTAPE 3 — Pièce d'identité + Photo
    Champ                    Type      Validation                Requis
    ─────────────────────────────────────────────────────────────────
    Type pièce              select    CNI/PASSEPORT/PERMIS      OUI
    Numéro pièce            text      6-20 caractères           OUI
    Date délivrance         date                                OUI
    Date expiration         date      > date délivrance         OUI
    Photo                   file      max 2MB, jpg/png          OUI
    Scan recto              file      max 5MB, jpg/png/pdf      OUI
    Scan verso              file      max 5MB                   NON

  VALIDATIONS MÉTIER :
  - Téléphone doit être unique (vérification API)
  - Email doit être unique si fourni
  - Âge minimum 18 ans
  - Pièce d'identité non expirée
  - Scoring anti-fraude avant enregistrement

  ACTIONS :
  - [Enregistrer] → POST /api/clients → 201 Created
  - [Annuler] → retour vers /clients
  - [Importer CNI] → scan puce (fonctionnalité future)

  ÉTATS :
  - LOADING : spinner pendant validation API
  - VALIDATION_ERROR : erreurs inline par champ + résumé en haut
  - API_ERROR : message d'erreur API (timeout, conflit, rate limit)
  - SUCCESS : redirection vers fiche client + notification toast


9. ÉCRAN CRITIQUE : MAKER-CHECKER VALIDATION
─────────────────────────────────────────────────

  Route : GET /validations → liste actions en attente
  Route : GET /validations/{id} → détail action
  Route : POST /validations/{id}/approve → approbation
  Route : POST /validations/{id}/reject → rejet

  DONNÉES PAR ACTION :
  - Type d'action (création client, blocage compte, décision crédit...)
  - Résumé des données soumises (JSON formaté)
  - Maker (nom, date de soumission)
  - Date d'expiration (T+48h auto-rejet si non traité)
  - Lien vers la ressource concernée (si existante)

15. CALLBACKS JAVASCRIPT POUR L'API
──────────────────────────────────────
  Les controllers Symfony ne génèrent pas les vues Twig directement
  à partir de l'API Java. Le flux est :

  1. Controller reçoit la requête HTTP
  2. Controller appelle ApiClientService (appel API Java)
  3. Controller vérifie le format de réponse
  4. Controller rend la vue Twig avec les données

  Exemple ClientCreateController::index (POST) :
  ┌────────────────────────────────────────────────────────────┐
  │ $apiClient->post('/api/clients', $formData);              │
  │ if ($response->getStatusCode() === 201) {                 │
  │     $this->addFlash('success', 'Client créé');            │
  │     return $this->redirectToRoute('client_detail',        │
  │         ['id' => $response->getData()['idClient']]);      │
  │ }                                                         │
  │ if ($response->getStatusCode() === 202) {                 │
  │     $this->addFlash('info', 'Action soumise (M-C)');     │
  │     return $this->redirectToRoute('validation_queue');    │
  │ }                                                         │
  │ if ($response->getStatusCode() === 409) {                 │
  │     $this->addFlash('error', 'Conflit: ' .               │
  │         $response->getData()['message']);                 │
  │     return $this->redirectToRoute('client_create');       │
  │ }                                                         │
  └────────────────────────────────────────────────────────────┘
